# Agent Documentation

This directory stores agent-facing documentation for APB implementation design, template behavior, CLI behavior, and workflow conventions.

Root `docs/` is optional and should only be used for human-facing framework documentation.

For broad orientation, start from [Agent Memory Index](../index.md). Documentation notes should link back to related planning notes, architecture decisions, review history, and template files when those links help future agents trace context.

## Template Documents

- Generated projects include `.agent/docs/code-organization.md` to define framework-first structure mapping, the recommended baseline source layout, module boundaries, shared code rules, and reuse checks.
