# Project Rules

## Language Policy

- Repository artifacts must be written in English unless explicitly approved otherwise.
- Communication with the project owner may use the owner's requested language.
- File names, configuration keys, commands, and technical terms should remain in English.

## Boundary Policy

- Keep work within the repository scope.
- Do not add application behavior, tooling, automation, or dependencies unless explicitly approved.
- Do not expand scope beyond the approved task.
- Preserve user-owned changes.

## Execution Policy

- Start from approved intent.
- Make the smallest useful change.
- Report created or modified files.
- Stop and wait for review when the task is complete.
